# KYROX Task System

Each implementation task must be written as a separate markdown file.

Recommended format:

```md
# Task ID: 0001
# Title:

## Goal

What should be achieved?

## Context

Why is this needed?

## Scope

Files/modules that may be changed.

## Out of Scope

What must not be changed.

## Acceptance Criteria

- [ ] Criterion 1
- [ ] Criterion 2

## Required Tests

- [ ] Unit test
- [ ] Integration test
- [ ] Architecture test

## Cursor Instructions

Specific implementation instructions for Cursor.

## CTO Review Focus

What the CTO should carefully review.
```

## Branch Naming

Use:

```text
feature/<task-id>-short-name
fix/<task-id>-short-name
refactor/<task-id>-short-name
test/<task-id>-short-name
```
